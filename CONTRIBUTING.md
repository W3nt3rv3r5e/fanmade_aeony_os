# Contributing to AeonyOS: Zenith

First off, thank you for considering contributing to the **AeonyOS** ecosystem. As a conceptual and fan-made project, we prioritize high-performance logic and a sharp, industrial aesthetic.

## 🛠 Architectural Principles
Before submitting a Pull Request, ensure your code adheres to these core standards:
* **Purged Logic:** Every component must be modular and free of "bloatware" or unnecessary dependencies.
* **Sharp Industrial Design:** All UI elements must maintain a `0px` border radius—rounded corners are strictly prohibited.
* **Neon Violet Aesthetic:** Use the global `--primary-color` variable for all interactive elements to ensure compatibility with the Theme Randomizer.
* **Moga Cursor:** Ensure custom interactions do not override the default rodless triangle cursor.

## 🚀 How to Contribute

### 1. Reporting Bugs
* Check the current repository issues to ensure the bug hasn't been reported.
* Provide a clear description of the performance lag or visual inconsistency.
* If the bug involves the **Zenith** boot sequence or startup audio, include your system hardware specs.

### 2. Feature Requests
* Features should focus on system-level simulation (e.g., Terminal improvements, File Explorer logic, or new Hex-based theme tools).
* Avoid suggesting third-party "Hub" integrations unless they fit the current modular architecture.

### 3. Pull Requests
1. Fork the repository and create your branch from `main`.
2. Ensure your code is formatted as a "Half-CSS, Half-JS" modular `.jsx` or a merged `index.html` structure.
3. Update the `README.md` if your change adds a new user-facing command.
4. Issue the PR with a clear summary of how it improves the Architect's vision.

## ⚠️ Anti-Fraud Policy
* **Originality:** All contributions must be original or under the MIT license.
* **Redistribution:** Contributors agree that their code becomes part of the official AeonyOS distribution. Unauthorized re-uploads of this project by "frauds" will be met with takedown requests.
